1. Who You Are

You are a Stores User — warehouse or stores staff who handle the physical movement of stock. Your user account has the Can Access Stores flag enabled, which unlocks the Stores section of the sidebar.

1.1 What you can do

  • Create a Goods Receipt Note (GRN) when suppliers deliver.
  • Create Issue Slips when goods leave the warehouse for a department or site.
  • Create Picking Slips when stock moves between storage locations (no approval).
  • See live stock levels in the Stock Register.
  • Manage the Stock Items catalogue.
  • Print PDF versions of any of your documents.

1.2 What you cannot do

  • You cannot approve GRNs or Issue Slips — that is Finance.
  • You cannot create Purchase Requisitions on behalf of other departments. Those are initiated by the department themselves.
  • You cannot delete a GRN or Issue Slip once submitted. If wrong, ask Finance to reject it.

1.3 Your responsibilities

  1. Record exact quantities on GRNs — these drive the stock register.
  2. Match deliveries to the right PR so spend is attributed to the correct department.
  3. Check the GRN is approved before issuing any stock (the system blocks you anyway).
  4. Keep the Stock Items list tidy — no duplicate descriptions.
  5. Promptly create picking slips when stock physically moves so the register stays accurate.

2. Quick Reference Card

One-page cheat sheet

Create GRN: Sidebar → Stores → Create GRN.

Create Issue Slip: Sidebar → Stores → Create Issue Slip.

Create Picking Slip: Sidebar → Stores → Create Picking Slip.

Print PDF: open the document → Download PDF.

Stock flow at a glance

  1. Goods arrive → you create GRN (pending approval).
  2. Finance approves GRN → stock is officially in register.
  3. Site requests goods → you create Issue Slip (pending HOD → pending Finance).
  4. Finance approves Issue Slip → stock is deducted.
  5. You physically hand over the goods to the issued-to person.

Checks before creating an Issue Slip

  • GRN exists and is approved.
  • You know the reference GRN number.
  • Customer field matches if the GRN is customer-reserved.
  • Available stock covers the requested quantity (system blocks over-issue).

3. Stores Workflow Diagrams

3.1 GRN flow (goods coming in)

[Supplier delivers] --> [You create GRN] --> pending_approval | Finance / assigned approver | approve / reject v approved (stock appears in register)

3.2 Issue Slip flow (goods going out)

[Site requests goods] --> [You create Issue Slip] --> pending_hod | HOD approve / reject v pending_finance | Finance approve / reject v approved (stock officially deducted)

3.3 Picking Slip flow (stock moving internally)

[Move goods between locations] --> [You create Picking Slip] --> completed (no approval)

3.4 How the Stock Register is computed

The Stock Register recalculates on every view using this simple formula per item:

Available = (sum of Quantity Received on approved GRNs) - (sum of Quantity Issued on approved Issue Slips) - (sum of Quantity on pending Issue Slips, shown as Reserved)

5. Task 1 — Create a Goods Receipt Note (GRN)

Use a GRN whenever goods physically arrive at the warehouse from a supplier.

5.1 Prepare before you start

  1. Check the delivery note from the supplier — this is your source of truth.
  2. Count the actual goods in the delivery. Note any damage, missing items, or over-deliveries.
  3. Find the PR number the delivery relates to (usually on the PO or delivery note).

5.2 Open the GRN form

  1. Sidebar → Stores → Create GRN.
  2. The form opens in a new page.
Create GRN form — header with PR reference and supplier, item table with Quantity Received column.

5.3 Fill in the header

  1. Reference PR — pick the approved purchase requisition from the dropdown.
  2. Supplier / Vendor — confirm the vendor matches what's on the PR.
  3. Delivery Date — actual date the goods arrived.
  4. Delivery Note Number — supplier's document number (useful for tracing).
  5. Customer — only fill this if the stock is reserved for a specific customer (e.g. project stock). Leave blank for general stock.

5.4 Record the items received

  1. For each line on the delivery:
    • Item Name / Description
    • Item Code (if known; pick from Stock Items dropdown)
    • Quantity Received — the actual count, not the ordered count
    • Unit
    • Condition note if any damage (write in description)
  2. If you received fewer items than ordered, record the actual received quantity. The difference remains on the PR for a later partial delivery.
  3. If you received extras, only record up to the PR quantity. Escalate the excess to Procurement.

5. Task 1 — continued

5.5 Submit the GRN

  1. Click Submit.
  2. The GRN is created with an ID like KSB-GRN-<timestamp>.
  3. Status becomes pending_approval.
  4. Finance (or the assigned approver) receives an email.

5.6 After submission

  1. The goods physically exist in the warehouse but are NOT yet shown as available in the Stock Register.
  2. Once Finance approves, stock becomes available and issue slips can draw from it.
  3. If Finance rejects (e.g. quantities don't match), the GRN is marked rejected. Create a new, corrected GRN.
Do not issue stock against a GRN that is still pending_approval. The system will block you, but more importantly, you'll be handing out stock that hasn't been formally accepted by Finance.

5.7 Common GRN mistakes

  1. Wrong PR reference — always double-check this, as it determines which department is charged.
  2. Quantity wrong — counting error. Re-count before submitting.
  3. Missing damaged-goods note — if 10 items arrived but 2 were damaged, record Quantity Received = 8 and note the damage in description.
  4. Blank customer field on customer-reserved stock — if the goods are for a specific customer, always fill this. Forgetting means someone else could issue the stock.

6. Task 2 — Create an Issue Slip

Use an Issue Slip whenever approved stock leaves the warehouse to go to a department or site.

6.1 Prepare before you start

  1. Confirm the person requesting has the authority to ask for the items.
  2. Know the destination (site, department, delivery address).
  3. Know the GRN reference number if drawing from a specific receipt.
  4. Check the Stock Register to confirm availability (Task 4).

6.2 Open the Issue Slip form

  1. Sidebar → Stores → Create Issue Slip.
  2. The form opens in a new page.
Create Issue Slip form with Issued To, Department, Reference Number (GRN), Customer, and Items table.

6.3 Fill in the header

  1. Issued To — the person receiving the goods.
  2. Department — the receiving department.
  3. Delivery Location — where to take the goods.
  4. Delivery Date — when the goods should arrive.
  5. Delivered By — the driver / runner / courier name.
  6. Reference Number — the GRN these items came in on (required if drawing from a specific GRN; leave blank only for exceptional direct-issue cases).
  7. Customer — must match the GRN customer if the GRN is customer-reserved.
  8. Remarks — any special handling instructions.

6.4 Add items

  1. Click Add Item.
  2. For each item:
    • Item Name / Description — must match what's on the GRN
    • Item Code
    • Quantity — cannot exceed remaining on the GRN
    • Unit
  3. The system checks availability as you type. If you exceed remaining stock, you see an error: Insufficient stock for "Widget": requested 10, available 7 (received 20, already issued 13)
The system prevents over-issues even if two users try to issue at the same moment. One will succeed; the other will get an error. This is deliberate — it stops stock from going negative.

6. Task 2 — continued

6.5 Submit the Issue Slip

  1. Click Submit.
  2. ID format: KSB-ISS-<timestamp>.
  3. Status becomes pending_hod.
  4. The HOD of the receiving department will approve first, then Finance.
  5. Stock is shown as Reserved on the register until Finance approves.

6.6 Hand over the goods

  1. Wait for status to become approved (HOD + Finance have both approved).
  2. Print the PDF of the Issue Slip (Task 6).
  3. Physically hand the goods to the delivery runner with the printed slip.
  4. Get the recipient to sign the printed slip on receipt. File the signed copy.
Some installations let you hand over goods immediately while the Issue Slip is still at pending_hod. Check your local policy. In any case, do not hand over goods if the HOD has rejected.

7. Task 3 — Create a Picking Slip

Use a Picking Slip when stock moves between storage locations within the warehouse (e.g. main store to site store). No approval needed.

7.1 Open the Picking Slip form

  1. Sidebar → Stores → Create Picking Slip.

7.2 Fill in the form

  1. Picked By — your name or the picker's name.
  2. Destination — where stock is going.
  3. Delivery Location — detailed address or bay number.
  4. Department — the department that owns the stock (not necessarily the receiver).
  5. Reference Number — optional link to a PR, GRN, or external job.
  6. Customer — if the stock is customer-reserved.
  7. Remarks — any handling notes.

7.3 Add items

  1. Click Add Item.
  2. For each item: name, code, quantity, unit.

7.4 Submit

  1. Click Submit.
  2. ID format: KSB-PCK-<timestamp>.
  3. Status: completed immediately — no approval required.
  4. Physically move the stock.
  5. Print the PDF and file for records.
A Picking Slip does NOT deduct stock from the register — it only records an internal movement. If stock is actually leaving the warehouse to a department, use an Issue Slip instead.

8. Task 4 — Use the Real-Time Stock Register

8.1 Open the Stock Register

  1. Sidebar → Stores → Real-Time Stock Register.
  2. The page shows every stock item with its current availability.

8.2 Understand the columns

  1. Item Code / Description — what it is.
  2. Stock In — total received on all approved GRNs.
  3. Stock Out — total issued on all approved Issue Slips.
  4. Reserved — quantity on pending (not yet approved) Issue Slips. Treated as unavailable for new issues.
  5. Available — the number you can still issue: Stock In − Stock Out − Reserved.

8.3 Search and sort

  1. Type in the Search box to filter by item description.
  2. Click any column header to sort.
  3. Sort by Available ascending to see low-stock items first.

8.4 Check a specific item before creating an Issue Slip

  1. Search for the item in the Stock Register.
  2. Note the Available quantity.
  3. Plan your Issue Slip quantity accordingly.
  4. If Available is less than the requested amount, you'll need to either wait for a new GRN or split the issue across multiple GRNs.
Stock Register showing ~30 items with Stock In, Stock Out, Reserved, and Available columns.

9. Task 5 — Manage Stock Items

Stock Items is the catalogue of known part codes and descriptions used to populate dropdowns on GRNs and Issue Slips.

9.1 Open Stock Items

  1. Sidebar → Stores → Stock Items.
  2. You see the full catalogue with code, description, unit, and preferred vendor.

9.2 Add a new item

  1. Click Add Stock Item.
  2. Fill in:
    • Item Number — unique code (e.g. IT-LAP-001 for a specific laptop model).
    • Item Description — clear, searchable name (e.g. "Dell Latitude 5420 14-inch laptop").
    • Unit — default unit for this item (pcs, kg, box).
    • Packaging UOM — how it comes (each, pack of 10, case of 50).
    • Preferred Vendor — suggestion for future PRs.
  3. Click Save.

9.3 Edit an existing item

  1. Click the row.
  2. Change what you need. Codes should not change — if wrong, create a new item and deactivate the old.
  3. Save.

9.4 Bulk upload from Excel

  1. Click Download Template (or similar) to get the Excel template.
  2. Fill in rows — one per item.
  3. Click Upload from Excel.
  4. Review any error messages row-by-row and correct the Excel before re-uploading.
Don't create two items with nearly-identical descriptions. If you see "Cement 50kg bag" and "Bag of cement 50kg" you have a duplicate — stock will be split across both and the register will under-report availability. Use the bulk-upload or a quick search before adding.

10. Task 6 — Print PDFs of Your Documents

10.1 Open the document

  1. From the list page (Goods Receipt Notes, Issue Slips, or Picking Slips), click the row.
  2. The detail page opens.

10.2 Download PDF

  1. Click Download PDF at the top-right.
  2. Your browser downloads the file — named after the document ID.

10.3 Print and sign

  1. Print the PDF on the warehouse printer.
  2. Have the receiver sign where indicated.
  3. File the signed copy in your physical records.
  4. Scan and attach the signed copy back to the system if your local policy requires it.

11. Troubleshooting

11.1 "Insufficient stock for X" when I create an Issue Slip

  1. Open the Stock Register and check Available.
  2. Note that Reserved quantities (pending Issue Slips) reduce Available. If you see a pending Issue Slip holding stock you need, contact the initiator of that slip.
  3. Reduce your requested quantity or split across multiple GRNs.

11.2 My GRN was rejected — what now?

  1. Open the rejected GRN.
  2. Read the rejection comment.
  3. Create a new GRN with the correction. You cannot edit the rejected one.

11.3 Stock Register shows the wrong number

  1. Check that all recent GRNs are approved (pending ones don't count as Stock In).
  2. Check for pending Issue Slips holding stock as Reserved.
  3. If still wrong, contact IT — there may be a data issue.

11.4 "GRN X is reserved for customer Y" error

  1. The GRN was created with a Customer value. Stock can only be issued if the Issue Slip has the same Customer value.
  2. Either set the Customer on your Issue Slip to match, or draw from a different (unreserved) GRN.

11.5 I can't see the Stores menu at all

  1. Your account doesn't have the "Can Access Stores" flag.
  2. Ask your Administrator to enable it on your user profile.

11.6 I submitted a GRN but there's no approval happening

  1. Check the Assigned Approver field on the GRN detail — is someone named?
  2. Contact that person, or contact Finance directly.
  3. Admin can reassign if the original approver is unavailable.

11.7 I need to cancel an Issue Slip I just created

  1. Ask the HOD or Finance to reject it with a comment.
  2. Stock that was Reserved returns to Available immediately on rejection.
  3. You cannot delete it yourself.

12.3 Daily stores checklist

  • Create GRNs for every delivery received yesterday.
  • Create Issue Slips for every goods-out request received today.
  • File signed printed copies of approved Issue Slips.
  • Check the Stock Register for low items and alert Procurement if re-order is needed.
